## WORKFLOW

### Step 1: Draft Recommendation

Create comprehensive hedge recommendation with:
- Position being hedged
- Risk identified
- Market details
- Suggested trade size
- Risk/reward analysis
- Hedge thesis
- Pros/cons

### Step 2: Send for Approval

**Via Discord:** Post to #cron-outputs-2 with approval buttons  
**Via Email:** Send to sam@impactquadrant.info with CC to agent  
**Timeout:** 24 hours (auto-expire if no response)

### Step 3: Wait for Response

**If APPROVED:**
1. Execute Kalshi trade via API
2. Log trade details
3. Update active hedges file
4. Confirm execution

**If REJECTED:**
1. Log rejection reason
2. Update recommendation with reason
3. Mark as "declined"
4. Learn for future (track why)

**If MODIFY requested:**
1. Adjust position size as requested
2. Re-calculate payouts
3. Re-send for approval

**If NO RESPONSE in 24h:**
1. Auto-expire recommendation
2. Log as "expired"
3. Do not execute

### Step 4: Post-Execution

- Log trade to `hedgemaster/trades/YYYY-MM-DD.json`
- Update `hedgemaster/active-hedges.json`
- Schedule settlement tracking
- Confirm via Discord/email
